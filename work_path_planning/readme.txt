global_planning：全局规划阶段。山地的无人车规划分为全局规划阶段和局部规划阶段。全局规划阶段使用了Voronoi的概念，首先对数据进行障碍判断，然后构建Voronoi图（具有边界的Voronoi图），还需要做点匹配问题（使用锚框概念缩短点匹配时间）
path_people：人员规划问题。由于人员具有高灵活度，所以在规划时只需考虑坡度对人员的影响。使用A*算法进行规划，将边缘设置为障碍。使用区域较大时，分块取数据后需要将数据拼接，将边缘设置为障碍方便拼接。
pg_plug：postgreSQL数据库的插件边界方法，目前以实现对输入某数据进行算法运算。无法实现从数据库中提取数据进行运算。
pro_treatment：对数据的预处理。该数据已是栅格状态，需要对其进行缺失值处理
test：为测试文件夹，在最开始熟悉数据是使用
Use_with_databases：对postgreSQL数据库进行连接操作。从数据库中提取固定格式数据，划分栅格后进行按照固定格式保存数据
